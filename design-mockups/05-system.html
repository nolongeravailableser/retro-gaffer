Why a system page is part of the proposal

  • Most of the friction in the current UI isn't any single screen — it's that colour, size and number-encoding rules are inconsistent between screens (OVR in the market but not the squad; green meaning brand, money-positive, active, and chemistry; three different price-button colours with no key). Codifying ~8 rules fixes dozens of small papercuts at once and keeps future features consistent.
  • The IA map shows where the demoted screens go — nothing is deleted, the 7-tab row just stops competing with the core loop.

Design system — colour roles, type, and the rules

Same retro-CRT identity, but every colour gets ONE job. The neon green stops being wallpaper and starts being a signpost.

Colour roles

--act #39FF14The primary action + active nav. Nothing else. One glowing thing per screen.
--cash #FFB000Money, only money. Never warnings (warnings use --warn).
--good #4ADE80Positive outcomes: wins, chem links, promotion zone. Softer than --act so it can't be mistaken for a button.
--warn #FBBF24Caution: out of position, window closing, board grumbles.
--bad #F87171Losses, bans, relegation zone, destructive actions.
--info #7DD3FCNeutral information, links, help.
--surface-1/2/3Three elevation steps instead of one near-black — cards can layer without borders doing all the work.

The eight rules

1 · One number per player: OVR, everywhere. Tier-coloured badge (87 green → 58 red). ATK/DEF and the 8 sub-stats live in the player sheet, one tap away — identical sheet in squad, market, draft and match.
2 · Never colour alone. Every status is icon/letters + colour: BAN INJ·2 TIRED. Promotion/relegation zones get the bar and a legend. Price tiers get words on the button (“Sign · Free”, “Poach · £39M”).
3 · 12px floor for data, 14px for body. The current UI sets stakes math, wage budgets and promotion rules at 9–11px. If it matters to a decision it gets ≥12px; if it doesn't, it goes behind an expander instead of shrinking.
4 · Numbers earn their place by being comparable. Raw “ATK 643 · DEF 531” for an opponent is banned — strength is always shown relative to you (edge bar + verdict in words).
5 · Explainer-at-the-point-of-use, not tooltips. Hover tooltips are dead weight on touch. Each rule of the game gets one inline sentence where it applies (segment subtitles, “How is this calculated?” expanders), then stays out of the way.
6 · One glowing CTA per screen. Primary = filled neon green. Secondary = outlined. Tertiary = ghost. Destructive = outlined red, never adjacent to the primary, always two-step.
7 · Stable chrome. The four tabs never appear/disappear per mode — a mode that doesn't support a tab shows it locked, with the reason. Modal stack is capped at one game-layer (decisions) above one screen-layer.
8 · Motion is feedback, not atmosphere. Score pops, payout count-ups, promotion confetti stay. The permanent logo flicker goes (it taxes attention all session); scanlines stay as static texture. All motion behind prefers-reduced-motion.

Information architecture — 7 mutating tabs → 4 stable ones

HOME next fixture · edge bar · stakes · wager · table snippet · week feed (absorbs Season + Inbox + JourneyBar) SQUAD pitch · bench · roster (OVR rows) · player sheet · training segment (absorbs Tactics + TrainingPanel) MARKET free agents / open market / at rivals · negotiation (locked state in Draft, with reason, instead of vanishing) CLUB career hub · records & trophies · challenges/scenarios · PvP & leaderboard · settings · save codes badge = inbox-style attention (board message, job offers) ☰ MENU new game · daily gauntlet · how to play · sound (destructive + occasional actions leave the header) LOGO → start menu (unchanged — it's good)

Alternative considered: keeping Records/Challenges/Compete as top-level tabs (status quo — they're discoverable there). Rejected: during an actual run those three are visited a fraction as often as Squad/Market/Home, and the 7-tab row is the reason the mobile nav needs 9px labels. Discoverability is preserved by Club-tab badges and the Start Menu's Records entry.
